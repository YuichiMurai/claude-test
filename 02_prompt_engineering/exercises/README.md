## 練習問題1: カスタムアシスタントの作成

### 目的
システムプロンプトを設計して、特定の用途に特化したアシスタントを作成する

### 要件
- 「料理レシピアドバイザー」のシステムプロンプトを設計する
- アシスタントは以下のペルソナを持つこと:
  - プロの料理人として振る舞う
  - 常に材料リスト・手順・調理時間を明記する
  - 初心者向けのアドバイスを含める
  - アレルギー情報を必ず確認する
- 複数のレシピリクエストをテストして一貫した応答を確認する

### 期待される出力
```
==============================
料理レシピアドバイザー
==============================

質問: 簡単なパスタレシピを教えてください

回答:
【料理名】ガーリックパスタ
【調理時間】20分
【材料】（2人前）
- パスタ: 200g
- ニンニク: 3片
...
【手順】
1. ...
2. ...
【初心者へのアドバイス】...
【アレルギー情報】小麦、（卵なし）
```

### ヒント
- システムプロンプトで出力形式を明確に定義する
- ペルソナの設定で一貫した口調を指定する
- 複数の質問をテストして一貫性を確認する

## 練習問題2: Few-shot でのテキスト分類

### 目的
Few-shot learning を使って独自の分類システムを構築する

### 要件
- ニュース記事を以下の5カテゴリに分類するシステムを作成する:
  - `tech`（テクノロジー）
  - `politics`（政治）
  - `sports`（スポーツ）
  - `business`（ビジネス・経済）
  - `entertainment`（エンタメ）
- 各カテゴリに2つ以上の Few-shot 例を用意する
- 10件のテスト記事を分類して正確性を確認する

### 期待される出力
```
==============================
ニュース分類システム（Few-shot）
==============================

記事: 新型AIチップが発表され、処理速度が従来の3倍に
分類: tech

記事: 日本代表がワールドカップ予選を突破
分類: sports

記事: 大手IT企業が小売チェーンを5000億円で買収
分類: business
...

正解率: X/10
```

### ヒント
- 各カテゴリの特徴的なキーワードを含む例を選ぶ
- 境界が曖昧なケースも例に含める（例: テク企業の経済ニュース）
- Few-shot の例の順序も結果に影響することがある

## 練習問題3: 複雑な推論タスク

### 目的
Chain of Thought と XML タグを組み合わせて複雑な推論タスクを解く

### 要件
- 以下の「プロジェクト優先度決定システム」を実装する:
  - 複数のプロジェクト情報を XML タグで構造化して入力する
  - CoT プロンプティングで各プロジェクトを評価させる
  - 評価基準: ビジネスインパクト、実現可能性、緊急度
  - 優先度スコアを算出して順位付けする
- 3〜5件のプロジェクトデータでテストする

### 期待される出力
```
==============================
プロジェクト優先度評価システム
==============================

評価プロセス:

【プロジェクトA: 顧客ポータルのリニューアル】
思考ステップ1: ビジネスインパクトを評価...
思考ステップ2: 実現可能性を評価...
思考ステップ3: 緊急度を評価...
スコア: 85/100

【プロジェクトB: 社内ツールの自動化】
...

【優先度ランキング】
1位: プロジェクトA（スコア: 85）
2位: プロジェクトB（スコア: 72）
...
```

### ヒント
- `<project>` タグにプロジェクト情報を構造化する
- システムプロンプトで評価基準とスコアリング方法を定義する
- CoT の指示で各評価ステップを明示する
- 最後に JSON 形式でスコアを出力させると集計しやすい
