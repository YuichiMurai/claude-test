# シナリオ3: コンテンツ分類・モデレーションシステム

**難易度:** ★★★★★  
**制限時間の目安:** 90分  
**配点:** 100点

---

## シナリオの説明

あなたはソーシャルプラットフォーム「ConnectHub」のエンジニアです。コンテンツ安全チームから、ユーザー投稿コンテンツを自動で分類・モデレーションするシステムの構築を依頼されました。

このシステムは以下の機能を提供します：
- 投稿に含まれる不適切コンテンツ（暴力・ヘイト・スパム等）を検出する
- 投稿のカテゴリ（テクノロジー・スポーツ等）を自動分類する
- 各判定に対する信頼度スコアを算出する
- 複数の投稿をバッチで効率的に処理する
- 結果を構造化 JSON 形式で出力する

---

## 詳細な要件

### 要件1: 不適切コンテンツの検出（必須）

投稿に不適切なコンテンツが含まれているかを検出してください。

検出対象と検出レベル（`high` / `medium` / `low` / `none`）：

| 不適切コンテンツの種類 | 説明 |
|---------------------|------|
| `violence` | 暴力的な表現、危害を加えることを煽るコンテンツ |
| `hate_speech` | 特定の属性（人種・性別・宗教等）を標的にした差別的表現 |
| `spam` | 無関係な宣伝・繰り返し投稿・フィッシング等 |
| `adult_content` | 性的に露骨なコンテンツ |

**出力形式（`dict`）:**
```python
{
    "violence": "none",        # high / medium / low / none
    "hate_speech": "none",
    "spam": "low",
    "adult_content": "none",
    "is_inappropriate": False, # いずれかが medium 以上なら True
    "details": "スパムの可能性があります"
}
```

### 要件2: カテゴリ自動分類（必須）

投稿のトピックカテゴリを自動分類してください。

**対応カテゴリ:**
- `technology` — テクノロジー・IT・プログラミング
- `sports` — スポーツ・フィットネス・アウトドア
- `entertainment` — エンターテイメント・映画・音楽
- `news` — ニュース・時事問題・政治
- `food` — 食事・料理・グルメ
- `travel` — 旅行・観光・体験
- `education` — 教育・学習・スキル
- `other` — 上記に該当しない

**複数カテゴリへの対応（マルチラベル分類）:**
- 1つの投稿が複数カテゴリに属せる（例: 「旅先でのグルメ体験」→ `travel` + `food`）
- 信頼度 0.5 以上のカテゴリを複数返すこと

### 要件3: 信頼度スコアの算出（必須）

各判定結果に対する信頼度（確かさ）を算出してください。

- スコア範囲: **0.0 〜 1.0**
- 各カテゴリ分類と不適切コンテンツ判定に対して個別にスコアを返す
- **閾値**: 0.5 以上を「判定あり」として扱う
- Claude に JSON 形式で返答させて数値を取得すること

### 要件4: バッチ処理対応（必須）

複数のコンテンツを一括で処理してください。

- `batch_process()` 関数で投稿リストを受け取り、全件処理する
- **レート制限対策**: API 呼び出し間に適切な待機時間（0.5秒以上）を設ける
- 処理済み件数・エラー件数をコンソールに出力する
- 1件エラーが発生しても残りの処理を継続すること

### 要件5: 結果の JSON 出力（必須）

処理結果を構造化 JSON 形式でファイルに保存してください。

**出力 JSON の構造:**
```json
{
    "metadata": {
        "processed_at": "2025-01-01T12:00:00",
        "total_count": 10,
        "inappropriate_count": 2,
        "categories_found": ["technology", "sports"]
    },
    "results": [
        {
            "id": "post_001",
            "text": "投稿テキスト（先頭50文字）...",
            "moderation": {
                "violence": "none",
                "hate_speech": "none",
                "spam": "none",
                "adult_content": "none",
                "is_inappropriate": false,
                "confidence": 0.95
            },
            "classification": {
                "categories": ["technology"],
                "confidence_scores": {
                    "technology": 0.92,
                    "education": 0.45
                }
            },
            "processed_at": "2025-01-01T12:00:01"
        }
    ]
}
```

## ヒント

- `ContentModerator` クラスに機能をまとめると整理しやすい
- モデレーションとカテゴリ分類を **1回の API コール** で行うと効率的（プロンプトで両方を依頼）
- JSON 形式のレスポンスには `json.loads()` を使用し、JSONDecodeError をキャッチすること
- バッチ処理のプログレス表示には `enumerate()` と `tqdm`（オプション）が使いやすい
- 信頼度スコアを Claude に生成させる場合、例示（Few-shot）があると精度が上がる
