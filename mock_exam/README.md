## 試験の進め方

### 基本的な流れ

1. **まずシナリオファイルを読む** — 要件と評価ポイントを把握する
2. **自分でコードを実装する** — Google Colab に新しいノートブックを作成して実装
3. **動作確認をする** — すべての要件が満たされているかテストする
4. **模範解答と比較する** — `solutions/` フォルダの解答と見比べて改善点を確認

### 制限時間の目安

- 各シナリオ: **90分**を推奨
- 合計: 約 270分（3シナリオ × 90分）

## 採点基準

各シナリオは **100点満点** で採点されます。

### 機能実装（40点）

| 評価項目 | 配点 | 評価基準 |
|---------|------|---------|
| 要件の充足度 | 20点 | 指定されたすべての要件を満たしているか |
| 正常動作 | 20点 | エラーなく期待通りに動作するか |

**ポイント:**
- 全要件を実装してから動作確認する
- エッジケース（空文字、長文など）でも動作するか確認する

### コード品質（30点）

| 評価項目 | 配点 | 評価基準 |
|---------|------|---------|
| PEP 8準拠 | 10点 | Pythonコーディング規約に沿っているか |
| 型ヒント | 10点 | すべての関数に型ヒントが付いているか |
| 適切なコメント | 10点 | コードの意図が伝わるコメントがあるか |

**ポイント:**
- 関数・クラスには必ず Docstring を書く
- 変数名は意味のある名前にする
- 複雑なロジックには説明コメントを追加する

### エラーハンドリング（15点）

| 評価項目 | 配点 | 評価基準 |
|---------|------|---------|
| 適切な例外処理 | 8点 | try/except で適切な例外をキャッチしているか |
| リトライロジック | 7点 | API エラー時に適切にリトライしているか |

**ポイント:**
- `except Exception:` でまとめてキャッチするのは避ける
- Anthropic の各種エラークラスを使い分ける
- リトライには Exponential backoff を使う

### パフォーマンス（15点）

| 評価項目 | 配点 | 評価基準 |
|---------|------|---------|
| トークン効率 | 8点 | 無駄なトークンを使わないプロンプト設計か |
| レスポンス速度 | 7点 | 処理速度を意識した実装か（キャッシュ・非同期など） |

**ポイント:**
- プロンプトは簡潔に、必要な情報のみを含める
- 同じ内容を繰り返し呼ぶ場合はキャッシュを検討する

## 学習のポイント

### 挑戦前の準備

1. **Phase 1〜4 の内容を復習する**
   - Phase 1: API の基本、マルチターン会話、ストリーミング
   - Phase 2: システムプロンプト、Few-shot学習、XML タグ
   - Phase 3: 入力検証、PII処理、エラーハンドリング、レート制限
   - Phase 4: キャッシング、非同期処理、モニタリング、スケーラブル設計

2. **各シナリオの要件を先に全部読む** — 実装前に全体像を把握する

3. **わからない場合は該当セクションに戻る**

   | 困っていること | 参照先 |
   |-------------|-------|
   | 会話履歴の管理 | `01_basics/04_message_history.py` |
   | システムプロンプト | `02_prompt_engineering/01_system_prompts.py` |
   | エラーハンドリング | `03_secure_applications/03_error_handling.py` |
   | レート制限 | `03_secure_applications/04_rate_limiting.py` |
   | キャッシング | `04_architecture/01_caching_strategy.py` |
   | 非同期処理 | `04_architecture/02_async_processing.py` |

### 模範解答の活用方法

1. **まず自分で実装する** — 模範解答を見る前に自分でチャレンジする
2. **比較して学ぶ** — 自分の実装と `solutions/` の模範解答を比較する
3. **違いを理解する** — なぜ模範解答がそのように実装されているか考える
4. **改善する** — 学んだことを自分のコードに反映する

### CCA 試験で重要なポイント

- **セキュリティ**: 入力検証、PII保護、プロンプトインジェクション対策
- **信頼性**: エラーハンドリング、リトライ、タイムアウト
- **スケーラビリティ**: キャッシング、非同期処理、バッチ処理
- **保守性**: 型ヒント、ドキュメント、テスト可能な設計
- **コスト最適化**: トークン効率、キャッシュ活用
